Self-Hosted Setup

Run Finlynq on your own infrastructure. App and PostgreSQL run in Docker; sensitive fields are encrypted at rest with a per-user key derived from your account password.

Quick Start with Docker Compose

Download the compose file:

curl -O https://raw.githubusercontent.com/finlynq/finlynq/main/docker-compose.yml

Create a sibling .env file with the three required secrets, each generated with openssl rand -hex 32:

echo "PF_JWT_SECRET=$(openssl rand -hex 32)" >> .env
echo "PF_PEPPER=$(openssl rand -hex 32)" >> .env
echo "PF_STAGING_KEY=$(openssl rand -hex 32)" >> .env

Then start it:

docker compose up -d

Open http://localhost:3000 and register. Compose fails fast with a clear message if any secret is missing. Change the default PostgreSQL password (POSTGRES_PASSWORD in .env) before exposing the container beyond localhost.

What You Get

  • Self-contained Docker Compose: app plus PostgreSQL, nothing external
  • All data stored in the postgres_data Docker volume
  • Envelope encryption on transaction text fields (payees, notes, tags, holdings)
  • First-party MCP server for plugging in AI assistants (Claude, etc.)
  • Automatic schema migrations on container start

Environment Variables

  • DATABASE_URL: PostgreSQL connection string (required; set for you by docker-compose)
  • PF_JWT_SECRET: JWT signing secret, at least 32 chars (required)
  • PF_PEPPER: scrypt password peppering, at least 32 chars (required in production)
  • PF_STAGING_KEY: wraps email-staged transactions, at least 32 chars (required in production)
  • PORT: server port (default: 3000)
